# المتطلبات المؤجَّلة للإنتاج — Tawjeeh HUB

هذا الملف يوثّق **كل شيء لا يمكن إكماله أو اختباره داخل بيئة التطوير الحالية**
لأنه يعتمد على مفاتيح/أسرار/خدمات خارجية يملكها صاحب المشروع. لا توجد أسرار هنا —
فقط أسماء المتغيّرات وأماكنها وأثرها.

> القاعدة: كل ما هو قابل للإصلاح بدون مفاتيح **تمّ إصلاحه**. كل ما يتطلّب مفتاحًا
> أو خدمة خارجية **مؤجَّل وموثَّق هنا**، ولا يُدّعى أنه «جاهز».

## كيف تقرأ الجدول

- **يحجب الاختبار؟** = هل يمنع غيابُه اختبارَ الميزة محليًا؟
- **يحجب بيعًا حقيقيًا؟** = هل يمنع غيابُه إتمام عملية شراء/تحميل فعلية في الإنتاج؟

## ملخّص ما يحجب «بيعًا حقيقيًا» (الحد الأدنى للإطلاق)

1. مشروع Supabase + تنفيذ `schema.sql` + المفاتيح الثلاثة.
2. رفع ملفات PDF خاصة + ربط `file_path`.
3. مزوّد دفع حقيقي: تنفيذ المحوّل + المفاتيح + `PAYMENT_WEBHOOK_SECRET` + تسجيل webhook.
4. `NEXT_PUBLIC_SITE_URL` الإنتاجي + نشر على الاستضافة.

كل ما عدا ذلك (Redis، Sentry، الدومين المخصّص) تحسينات لا تحجب البيع.

> راجع أيضًا: `DEPLOYMENT_CHECKLIST.md` (الخطوات)، `docs/SUPABASE_SETUP.md`
> (التفصيل)، `docs/KNOWN_LIMITATIONS.md` (القيود التقنية المعروفة).
