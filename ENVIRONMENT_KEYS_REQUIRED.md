# المفاتيح والخدمات الخارجية المؤجَّلة — مركز توجيه

كل ما يلي **مؤجَّل** ولا يوقف التطوير: المنصة تعمل الآن بالكامل في وضع Seed/Demo
(تخصصات + وظائف + مطابقة + توجيه) بدون أي مفتاح. هذا الملف يوثّق ما يلزم لاحقًا.

| المفتاح / الخدمة | من أين نحصل عليه | لماذا نحتاجه | أساسي / لاحق | ماذا يحدث إن لم يوضع |
| --- | --- | --- | --- | --- |
| `NEXT_PUBLIC_SUPABASE_URL` · `NEXT_PUBLIC_SUPABASE_ANON_KEY` · `SUPABASE_SERVICE_ROLE_KEY` | Supabase → Settings → API | الحسابات، حفظ الوظائف، تخزين جداول `majors`/`jobs`، المتجر | أساسي للحساب/الحفظ | المنصة تعمل بوضع Seed؛ تسجيل الدخول والحفظ معطّلان بلطف |
| `PAYMENT_PROVIDER` + مفاتيح المزوّد + `PAYMENT_WEBHOOK_SECRET` | Moyasar / HyperPay / PayTabs | بيع الأدلة المهنية فعليًا | لاحق | المتجر يعرض المنتجات؛ الشراء الحقيقي معطّل (mock) |
| مصادر الوظائف (RSS / API) | لوحات التوظيف الرسمية التي تتيح RSS أو API بشكل **مسموح** | سحب وظائف حقيقية بدل بيانات seed | لاحق | المنصة تعرض وظائف seed؛ تُضاف المصادر في `DEFAULT_SOURCES` |
| `JOBS_INGEST_TOKEN` (اختياري) | يُولَّد ذاتيًا | تأمين تشغيل خط السحب عبر cron/endpoint | لاحق | السحب يُشغَّل يدويًا فقط |
| مزوّد ذكاء اصطناعي (مثل مفتاح نموذج لغوي) | مزوّد AI معتمد | ترقية التوجيه الذكي (المهارات/الدورات) من قواعد إلى AI | لاحق | يعمل التوجيه بالقواعد الداخلية الحالية بدون AI |
| `UPSTASH_REDIS_*` | Upstash | rate limiting موزّع + جدولة | لاحق | يُستخدم محدِّد في الذاكرة لكل نسخة |

## مبادئ مهمة
- **لا مفتاح يوقف المنصة.** أي خدمة غير مهيأة → وضع Seed/Demo + رسالة لطيفة، لا 500.
- **لا scraping عدواني.** مصادر الوظائف يجب أن تكون قانونية (RSS/API/صفحات مسموح قراءتها/إدخال يدوي).
- **لا أسرار في الكود ولا في Git.** كل المفاتيح في إعدادات الاستضافة فقط.

## تفعيل التخزين الدائم (Phase 3 — Persistence)
الجداول `majors / jobs / saved_jobs / applications` صارت في الـschema مع RLS وفهارس.
لتفعيلها على قاعدة بيانات قائمة:
1. شغّل `supabase/migrations/0002_jobs_majors_saved_applications.sql` (الجداول + RLS).
2. عبّئ جدول التخصصات من المصدر الواحد:
   `npm run seed-majors` (يحتاج مفاتيح Supabase في `.env.local`).
3. الوظائف تُملأ لاحقًا عبر خط السحب (Phase 4). **حتى ذلك الحين يبقى المتجر
   والصفحات تعمل ببيانات seed تلقائيًا** — لا يلزم إيقاف.

> سلوك الـfallback: إذا كانت Supabase غير مهيأة، أو الجداول فارغة، أو حدث أي خطأ
> اتصال — تعرض المنصة بيانات seed بدل أن تتعطّل (لا 500/شاشة سوداء). الوظائف
> المحفوظة تُحفظ في المتصفّح للزائر، وتُزامَن مع حسابه تلقائيًا عند تسجيل الدخول.

## كيف تُفعّل السحب الحقيقي لاحقًا
1. أنشئ adapter جديدًا في `src/lib/jobs/ingestion/sources/` يطبّق `SourceAdapter`
   (legal source فقط).
2. سجّله في `DEFAULT_SOURCES` داخل `src/lib/jobs/ingestion/index.ts`.
3. فعّل `saveJobs` للكتابة في جدول Supabase `jobs` (يتطلب service role).
4. جرّب: `npm run jobs:ingest` (تشغيل تجريبي يطبع ما سيُحفظ).
