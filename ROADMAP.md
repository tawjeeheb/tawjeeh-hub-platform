## البنية التحتية والأمان
- [ ] ترقية rate limiting إلى Upstash Redis (موزّع عبر عدة نسخ).
- [ ] **ترقية Next.js إلى v16 لإغلاق تنبيهات npm audit المتبقية.**
      الإصدار الحالي هو أحدث مرقّع في سلسلة 14.x (`14.2.35`)، والتنبيهات المتبقية
      على مستوى إطار العمل وإصلاحها الوحيد هو `next@16` (ترقية كبرى كاسرة = بنية
      خارجية، لذلك أُجّلت). تحليل القابلية للتطبيق على هذا المشروع:
      - i18n / Pages Router bypass: غير مستخدم (App Router فقط) — لا ينطبق.
      - WebSocket upgrades SSRF: غير مستخدم — لا ينطبق.
      - CSP nonces XSS / beforeInteractive XSS: غير مستخدمين — لا ينطبق.
      - Image Optimizer DoS: `next/image` يُستخدم للشعار المحلي فقط؛ لا صور بعيدة
        تُخدَم حاليًا (`remotePatterns` مقيّد بـ `*.supabase.co` للمستقبل).
      - postcss المضمّن داخل next: لا إصلاح ضمن 14.2.x.
- [ ] مراجعة CSP وتضييق `script-src`/`style-src` باستخدام nonces.
- [ ] إضافة CAPTCHA على التسجيل/تسجيل الدخول عند الحاجة.
- [ ] لوحة تحليلات أساسية (مبيعات، تحويلات).
